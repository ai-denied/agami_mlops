## 감정 라벨 구조 개선

기존에는 EMOTIC 감정 라벨과 직접 수집한 감정 라벨 구조가 서로 달라 일관성 있는 학습이 어려운 문제가 있었다.

초기에는 단일 대표 감정으로 변환하는 단일 레벨 구조를 고려했지만, 실제 감정은 여러 감정이 동시에 섞여 나타나는 경우가 많다고 판단했다. 최종적으로 **멀티라벨 구조**가 상황형 CAPTCHA 목적에 더 적합하다고 판단하여 멀티라벨 기반 감정 구조를 유지하는 방향으로 수정했다.

---

## AI 공격 기반 CAPTCHA 검증 시스템 구축

OpenAI GPT-4o-mini Vision API를 활용해 이미지 입력 → 감정 추론 → 실제 라벨 비교 → 결과 저장 과정을 자동화했다. 추가적으로 confidence 값, 선택 이유, 혼동 가능 감정도 함께 저장하여 AI가 어떤 감정에서 혼동하는지 분석할 수 있도록 구성했다.

---

## 트러블슈팅 — Vision API 비용 및 요청 제한 문제

**원인**  
대량 이미지 검증 과정에서 Vision API 비용 증가, request quota 초과, connection error, rate limit 문제가 반복적으로 발생했다. 비용 절감을 위해 Gemini API 기반 검증도 시도했지만 무료 티어 환경에서는 quota/request 제한 문제가 계속 발생했다.

**해결 — 오픈소스 VLM 기반 로컬 추론 구조 전환**  
Google Colab GPU 환경에서 **Qwen2.5-VL-3B-Instruct** 기반 오픈소스 Vision-Language Model을 활용하는 방향으로 전환했다.

- API 비용 문제 감소
- request 제한 제거
- 대량 검증 가능

외부 API 의존도가 줄어들면서 향후 자동 CAPTCHA 평가 파이프라인 구축에도 더 유리하다고 판단했다.
