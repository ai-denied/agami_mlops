# 손전등 CAPTCHA 데이터셋 구축 마무리 회고록

**일자**: 2026-04-28  
**담당**: agami MLOps

---

## 배경

기존 CAPTCHA 방식들은 단순 이미지 분류 수준으로 해결 가능한 경우가 많다고 판단했다. 그래서 단순 정답 클릭이 아니라 **문제를 해결하는 과정 자체를 분석하는 방향**으로 프로젝트를 진행하기로 했다.

---

## 구현 내용

### 손전등 기반 CAPTCHA 데이터셋 구축

- 배경 이미지와 객체 PNG를 직접 수집
- 객체를 랜덤 위치/크기로 합성하는 코드 구현 → 다양한 상황 생성
- 합성과 동시에 bbox 라벨 자동 생성

**결과**
| 항목 | 내용 |
|------|------|
| CAPTCHA 이미지 | 약 1,000장 생성 |
| bbox 라벨 | 자동 생성 완료 |

이후 데이터셋에 대해 다음 검증을 진행하여 학습 가능한 형태로 정리했다.
- 이미지/라벨 개수 검증
- bbox 좌표 오류 검사
- train / valid / test 분리

### 마우스 행동 로그 시스템 구축

실제 사용자 행동 데이터를 수집하기 위해 Flask API 기반 로그 시스템을 구축했다.

기록 항목:
- 시간(t)
- 좌표(x, y)
- 클릭 위치
- success / fail 여부

JSON 형태로 저장하도록 구현. 단순 정답 여부보다 **사용자가 목표를 탐색하는 행동 자체**가 중요한 데이터가 될 수 있다고 판단했다.

---

## 고민했던 문제

손전등 영역 크기나 배경 대비에 따라 객체가 너무 쉽게 보이는 경우가 발생했다.

- 어떤 이미지는 객체가 배경과 자연스럽게 섞여 어려웠고
- 어떤 이미지는 색 대비 때문에 객체가 바로 눈에 띄었음

단순 랜덤 생성만으로는 **난이도 조절이 어렵다**는 문제를 느꼈다.
