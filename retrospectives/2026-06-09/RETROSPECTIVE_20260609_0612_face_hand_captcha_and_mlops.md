# 얼굴/손 기반 CAPTCHA 데이터셋 정리 및 MLOps 운영 구조 구축 회고록

**일자**: 2026-06-09 ~ 2026-06-12  
**담당**: agami MLOps

---

## 1. 얼굴/손 기반 CAPTCHA 데이터셋 정리 및 분석

실시간 행동 CAPTCHA 개발을 위해 얼굴 및 손 데이터셋 구조를 재검토했다.

AIHub 안면인식 보안 데이터셋을 다운로드하여 데이터 구조를 분석했으며, 실제 사용자 데이터(Live)와 다양한 공격 데이터(Print Attack, Replay Attack, 3D Mask Attack)를 구분하여 정리했다.

| 구분 | 수량 |
|------|------|
| 총 데이터 수 | 16,016장 |
| Real 이미지 | 8,866장 |
| Spoof 이미지 | 7,150장 |

손동작 데이터는 별도 모델 학습에 활용하기보다 **MediaPipe 기반 Rule 검증 방식**으로 활용하는 방향으로 설계를 변경했다.

---

## 2. 안면 인식 데이터 전처리 개선

영상 데이터와 이미지 데이터의 차이를 분석하면서 기존 전처리 방식의 한계를 확인했다.

기존 방식은 프레임 수가 영상마다 달라 시계열 Feature 계산 결과의 신뢰도 차이가 발생했고, 시계열 모델 학습에 필요한 변화량 Feature가 부족한 문제가 있었다.

**개선 내용**
- 시계열 데이터셋 구조 재설계: 프레임 시퀀스를 15프레임으로 고정
- 변화량 기반 Feature 추가, Feature 종류 확장
- 절대좌표 기반 Feature → 상대좌표 기반 Feature로 변경
  - 얼굴 중심 기준 상대좌표 변환
  - 얼굴 폭(Face Width) 기준 정규화 적용
  - 촬영 환경과 카메라 위치 영향 최소화

위치 정보 자체보다 사용자의 실제 움직임 패턴을 학습할 수 있는 형태로 Feature를 재설계했다.

## 3. MLOps 모델 운영 구조 구축

Flashlight CAPTCHA 모델 운영을 위한 MLOps 구조를 설계했다.

- **Model Store 구축**: 모델 저장 구조 정리, 모델 버전 관리 체계 수립
- **모델 패키징 자동화**: 학습 결과 자동 패키징, ONNX 모델 로딩 구조 정리
- **모델 비교 자동화**: 현재 운영 모델과 후보 모델 성능 비교 기능 설계, 모델 승격(Promotion) 조건 정의
- **재학습 정책 수립**: 재학습 기준 및 운영 정책 정의, Argo Workflow 기반 자동화 구조 초안 작성

## 트러블슈팅 — 절대좌표 기반 Feature의 환경 민감도 문제

**문제**  
기존 얼굴 Feature가 절대좌표 기반으로 설계되어 있어 카메라 위치나 얼굴 크기에 매우 민감했다. 동일 사용자라도 촬영 거리나 촬영 환경이 달라지면 Feature 분포가 크게 달라져 모델 일반화 성능이 저하될 가능성이 있었다.

**원인 분석**
- 랜드마크 좌표를 절대 위치 기준으로 사용
- 얼굴 크기 변화에 따른 좌표 값 변화 발생
- 카메라 위치 변화에 따른 Feature 편차 증가

**해결**
- 얼굴 중심 기준 상대좌표 변환
- 얼굴 폭(Face Width) 기준 정규화 적용
- 환경 정보 제거 후 움직임 패턴 중심 Feature 설계

**결과**  
촬영 환경 변화에 대한 영향을 줄이고 사용자 행동 패턴 자체를 반영할 수 있는 Feature 구조로 개선했다.
