# 새로운 상황형 CAPTCHA 설계 시작 회고록

**일자**: 2026-04-30  
**담당**: agami MLOps

---

## 방향 전환

기존 손전등 CAPTCHA는 객체 탐색 중심이었다면, 새로운 방향은 **상황 맥락 이해 기반 캡챠**였다.

단순히 보이는 것을 찾는 문제나 단순 감정 분류 문제가 아니라, **상황을 이해해야만 정답을 판단할 수 있는 캡챠**를 목표로 설계 방향을 전환했다.

---

## 데이터 수집 방식 실험

| 방식 | 장점 | 단점 |
|------|------|------|
| ChatGPT 수작업 이미지 생성 | 의도한 상황을 정확하게 생성 가능 | 이미지 하나씩 직접 생성해야 해서 매우 비효율적 |
| Pexels API 기반 자동 수집 | 무료, 빠른 대량 수집 가능 | 상황 다양성 제한적, 원하는 맥락 표현 부족 |
| OpenAI API 기반 자동 생성 | 상황 의도를 정확하게 반영, 자동 생성 가능 | 생성 속도 느림, API 비용 발생 |

---

## 논의했던 핵심 문제

현재 구조는 "사람이 이미지 생성 → 사람이 검수 → 사람이 저장" 형태로 진행되고 있어, 데이터 품질은 높지만 **생산성이 매우 낮았다.**

또 다른 핵심 문제는 "생성형 AI가 만든 이미지를, 역으로 AI가 충분히 해석할 수 있는 것 아닌가?"라는 점이었다. 결국 **"AI가 쉽게 푸는 캡챠라면 사람만 풀 수 있는 문제가 아니다"**라는 결론에 도달했다.

---

## 해결 방향

실제로 AI 모델을 공격자로 설정하고 검증하기로 결정했다.

1. AI에게 CAPTCHA 이미지 입력
2. 상황 판단 수행
3. 실제 정답과 비교
4. 정답률 측정

이를 통해 AI가 쉽게 맞추는 데이터는 제거하고, AI가 헷갈리는 데이터만 남기는 방식으로 캡챠 난이도 자체를 설계하는 방향을 고민하게 되었다.
