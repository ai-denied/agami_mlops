## ML 파이프라인 구조 설계

손전등 CAPTCHA를 기준으로 ML 파이프라인 구조를 설계했다. 기존에는 데이터 수집, 전처리, 모델 학습 코드가 다소 흩어져 있었기 때문에, 이를 하나의 학습 파이프라인으로 정리하는 방향을 잡았다.

전체 흐름:

```
Raw mouse log 수집 → Feature 추출(정적/동적) → 학습 데이터 병합
→ train/valid/test 분리 → 모델 학습 → 평가 → 추론 API 연결
```

GPU 서버의 역할도 다시 정리했다. 단순히 파일을 올리고 학습하는 서버가 아니라 데이터 전처리 / 모델 학습 / 모델 평가 / 모델 서빙을 단계별로 수행할 수 있는 구조로 바라보기 시작했다.

## 대시보드 구조 설계

서비스 운영 관점에서 필요한 대시보드 구조를 설계했다. 필요한 정보:

- 전체 인증 수
- 성공/실패 비율
- CAPTCHA 유형별 성공률
- AI 공격 탐지 수
- 위험도 높은 시도 목록
- 평균 인증 시간
- 재시도 비율
- 사용자 행동 로그 분석 결과

이를 기반으로 대시보드 API 명세서도 작성했다.

## Playwright 기반 CAPTCHA 매크로 테스트 시작

실제 공격자가 브라우저를 조작하는 상황을 가정하기 위해 Playwright 기반 매크로 테스트 코드를 작성했다.

구현한 봇 유형:
- 랜덤 탐색 봇
- 지그재그 탐색 봇
- 정답 좌표를 알고 있는 봇

이 작업의 목적은 단순히 CAPTCHA를 뚫는 것이 아니라, **현재 CAPTCHA 구조가 어떤 방식의 공격에 취약한지** 확인하는 것이었다.
