# Stable Diffusion 기반 자동 생성 환경 구축 회고록

**일자**: 2026-05-07  
**담당**: agami MLOps

---

## 목표

상황형 CAPTCHA 데이터셋을 대량으로 확보하기 위해 Stable Diffusion 기반 자동 생성 환경 구축을 시도했다.

---

## 트러블슈팅 1 — 로컬 환경 구축 실패

**원인**
- Python 3.12 호환 문제 (Stable Diffusion은 Python 3.10 권장)
- Torch / CUDA 충돌
- NVIDIA GPU 미지원 환경

**해결**  
로컬 환경보다 **Google Colab GPU 환경**이 훨씬 현실적이라고 판단하고 전환.

---

## Google Colab 기반 자동 생성 파이프라인 구축

Colab 환경에서 Diffusers / SDXL / RealVisXL 기반 자동 생성 구조를 구축했다.

파이프라인: 상황 템플릿 정의 → 랜덤 조합 → 프롬프트 생성 → 이미지 저장 → `metadata.csv` 기록 — 까지 자동화.

---

## 트러블슈팅 2 — 감정이 너무 직관적으로 드러나는 문제

생성된 이미지들을 확인해보니 상황 판단이라기보다 **단순 감정 분류처럼 보이는 문제**가 있었다.
- 표정만 봐도 정답이 너무 명확함
- AI가 얼굴만 보고 쉽게 예측 가능함

또한 공개 모델 기반 생성 특성상 얼굴 일그러짐, 디테일 부족, 부자연스러운 표정 문제도 자주 발생했다.

**해결 — EMOTIC 데이터셋 발견**  
관련 공개 데이터셋을 조사하여 EMOTIC 데이터셋을 발견했다. EMOTIC은 단순 얼굴 감정 데이터셋이 아니라 상황·행동·주변 맥락 정보까지 포함된 데이터셋이었다. 상황형 CAPTCHA의 참고 데이터셋으로 활용하기로 결정했다.
