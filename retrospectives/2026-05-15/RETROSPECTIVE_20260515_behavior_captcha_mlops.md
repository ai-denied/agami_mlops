## 방향 확장

프로젝트 방향이 "정적 이미지 CAPTCHA"에서 **"실시간 행동 기반 CAPTCHA"**로 확장되기 시작했다. 얼굴 인식 자체보다 "사람다운 행동 패턴" 검증 방향으로 구조를 설계했다.

예시 요소:
- 자연스러운 얼굴 움직임
- 반응 지연
- 랜덤 행동 흐름
- landmark 변화 패턴

## MLOps 아키텍처 설계 시작

AGAMI 전체 서비스 구조를 기준으로 MLOps 아키텍처 설계도 함께 진행했다. 기존에는 일반 웹서비스 배포 구조 중심이었다면, 이번에는 ML 학습, 모델 배포, 추론 API, 자동화 파이프라인까지 포함한 구조를 고민하기 시작했다.

**설계 방향**: 일반 서비스 CI/CD와 ML 파이프라인 분리

```
GitHub Actions → Container Registry(Harbor) → K3s 배포
```

**GPU 서버 역할 분리 고민**
- Model Training Pod — 모델 학습 전용
- Model Serving API Pod — 실시간 추론 전용

역할을 나눠 학습 중에도 서비스 안정성 유지, 추론 지연 감소, 리소스 충돌 최소화를 목표로 했다.

## 이번 주 회고

이번 주에는 단순 CAPTCHA 구현을 넘어 AI 공격 검증, 감정 데이터 구조 설계, 행동 기반 CAPTCHA 방향, 실시간 추론 구조, MLOps 아키텍처까지 프로젝트 범위가 크게 확장되었다.

가장 중요했던 부분은 **AI가 실제로 헷갈리는 데이터가 무엇인가**를 직접 검증하기 시작했다는 점이었다. 단순 이미지 분류가 아니라 상황 맥락, 행동 패턴, 실시간 반응까지 함께 고려해야 한다는 방향성이 점점 명확해졌다.
